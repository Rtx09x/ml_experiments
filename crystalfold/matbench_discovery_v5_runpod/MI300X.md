# MI300X Notes

This path is optimized for a first ROCm attempt without pretending the CUDA
defaults are automatically ideal.

## What changed

- `torch.compile` is intentionally avoided on ROCm by default
- dataloader `pin_memory` is disabled on ROCm
- TF32 CUDA-only tuning is skipped on ROCm
- BF16 remains the default mixed-precision mode

## Recommended approach

1. Download artifacts from HF.
2. Run a 1-epoch smoke test.
3. If that passes, scale batch size.

## Smoke train

```bash
/workspace/crystalfold_venv/bin/python -m crystalfold_v5.cli train \
  --dataset /workspace/crystalfold_v5_artifacts/mpt_v5.h5 \
  --run-dir /workspace/crystalfold_v5_artifacts/runs/mi300x_smoke \
  --epochs 1 \
  --batch-size 256 \
  --workers 8 \
  --amp bf16 \
  --no-pin-memory
```

## Full train

Start here:

```bash
/workspace/crystalfold_venv/bin/python -m crystalfold_v5.cli train \
  --dataset /workspace/crystalfold_v5_artifacts/mpt_v5.h5 \
  --run-dir /workspace/crystalfold_v5_artifacts/runs/mi300x_v5 \
  --epochs 80 \
  --batch-size 256 \
  --workers 8 \
  --amp bf16 \
  --no-pin-memory
```

If memory/utilization looks comfortable, try `--batch-size 384`, then `512`.
